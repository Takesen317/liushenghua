# 留声画 - 超级详细部署操作手册

**适合对象**：从未用过 GitHub、Vercel、Railway 的新手
**费用**：完全免费
**预计时间**：30 分钟

---

## 第一阶段：准备工作

### 第一步：确认代码位置

在电脑上找到你的项目文件夹，确认路径是：
```
F:\Code\test2
```

打开文件夹，确认里面有这些内容：
- `backend` 文件夹
- `frontend` 文件夹
- `docs` 文件夹
- `docker-compose.yml` 文件

---

## 第二阶段：注册 GitHub

GitHub 是一个存放代码的网站，类似于"代码的网盘"。

### 2.1 打开 GitHub 注册页面

1. 打开你的浏览器（推荐 Chrome 或 Edge）
2. 在地址栏输入：`https://github.com`
3. 按回车

**你看到的页面**：GitHub 首业，黑色背景。

### 2.2 点击注册按钮

1. 在页面右上角找到 **Sign up** 按钮（绿色按钮）
2. 点击它

### 2.3 填写注册信息

**在 Enter your email 框中**：
- 输入你的邮箱地址（建议用 QQ 邮箱或网易邮箱）

点击 **Create account**

**在 Create a password 框中**：
- 输入密码（至少 8 位）

点击 **Create account**

**在 Enter a username 框中**：
- 输入你想用的用户名（英文，比如 `liushenghua2024`）
- 这个会成为你的 GitHub 地址的一部分

点击 **Create account**

**在 Verification 那里**：
- 按提示完成验证（勾选或滑动等）

点击 **Create account**

### 2.4 验证邮箱

1. 打开你的邮箱，找到 GitHub 发的邮件
2. 点击邮件里的链接完成验证
3. 验证成功后，GitHub 会跳转到欢迎页面

---

## 第三阶段：创建 GitHub 仓库

仓库（Repository）就是存放代码的地方，类似于一个项目文件夹。

### 3.1 创建新仓库

**如果已经跳转到欢迎页面**：
1. 点击 **Create a repository** 按钮

**如果回到首页**：
1. 点击右上角的 **+** 按钮（在新回复的左边）
2. 选择 **New repository**

### 3.2 填写仓库信息

在创建仓库页面，填写以下内容：

**Repository name**（必填）：
- 输入：`liushenghua`
- 注意：全部用小写字母

**Description**（选填）：
- 输入：`留声画 - AI照片语音解说生成器`

**选择 Public**：
- 点击 **Public** 单选框
- Public 表示公开仓库（必须选这个）

**不要勾选**：
- 不需要勾选 "Add a README file"
- 不需要勾选 "Add .gitignore"

### 3.3 点击创建按钮

滚动到页面底部，找到 **Create repository** 按钮（绿色），点击它。

### 3.4 记录仓库地址

创建成功后，你会看到一个页面。
**找到这个地址并复制下来**：
```
https://github.com/你的用户名/liushenghua
```

---

## 第四阶段：上传代码到 GitHub

这是最关键的一步！按照下面的步骤一步一步来。

### 4.1 打开命令提示符

**方法一**：
1. 按键盘上的 **Windows键 + R**（Windows键是 Alt 左边的那个）
2. 输入 `cmd`
3. 按回车

**方法二**：
1. 在任务栏搜索框输入 `cmd`
2. 点击"命令提示符"

**你看到的**：一个黑色窗口，里面有白色文字。

### 4.2 检查 Git 是否已安装

在黑色窗口里输入（直接复制粘贴就行）：

```bash
git --version
```

按回车。

**如果显示类似以下内容**，说明 Git 已安装：
```
git version 2.43.0.windows.1
```

**如果显示"'git' 不是内部或外部命令"**，继续看下一节安装 Git。

---

### 4.2.1 安装 Git（如果上一步失败）

**如果 git 命令找不到，按以下步骤安装**：

1. 打开浏览器，访问：**https://git-scm.com/download/win**
2. 页面会自动下载 Git 安装包
3. 下载完成后，在下载文件夹里找到 `Git-xxx.exe` 文件
4. **双击运行它**

**安装向导设置**（全部选默认，一路点 Next）：

1. 欢迎页面：点 **Next**
2. 安装位置：默认就行，点 **Next**
3. 组件：默认就行，**勾选"Add to PATH"**，点 **Next**
4. 编辑器：默认 Vim 或者选 Notepad，点 **Next**
5. 之后的页面全部点 **Next**
6. 安装完成：点 **Finish**

**安装完成后**，重新打开命令提示符（关掉之前的，重新打开 Win+R → cmd）

---

### 4.3 切换到项目目录

在命令提示符里依次输入（每行输完按一次回车）：

**如果你的代码在 D 盘**：
```bash
d:
cd Code\test2
```

**如果你的代码在 C 盘**：
```bash
c:
cd Code\test2
```

**验证**：输入以下命令查看文件夹内容：
```bash
dir
```

**你应该看到**：
```
backend  docs  frontend  docker-compose.yml  ...
```

---

### 4.4 配置 Git 用户信息

在命令提示符里输入（**把邮箱和用户名换成你自己的**）：

```bash
git config --global user.email "你的邮箱地址"
git config --global user.name "你的GitHub用户名"
```

按回车。

---

### 4.5 初始化 Git 仓库

```bash
git init
```

按回车。

**你会看到类似这样的输出**：
```
Initialized empty Git repository in F:/Code/test2/.git/
```

---

### 4.6 添加所有文件

```bash
git add .
```

按回车。**注意**：add 后面有一个空格，然后是一个英文句号。

---

### 4.7 提交文件（创建版本）

```bash
git commit -m "first commit"
```

按回车。

**你会看到类似这样的输出**：
```
[main (root-commit) abc1234] first commit
 50 files changed, 1234 insertions(+)
```

---

### 4.8 连接 GitHub 仓库

**重要**：把下面的 `你的GitHub用户名` 替换成你刚才注册的用户名！

```bash
git remote add origin https://github.com/你的GitHub用户名/liushenghua.git
```

按回车。

---

### 4.9 推送代码到 GitHub

```bash
git branch -M main
git push -u origin main
```

按回车。

**第一次推送会要求登录**：
1. 会弹出一个窗口让你输入用户名和密码
2. 用户名：输入你的 GitHub 用户名
3. 密码：**输入你的 GitHub 密码**（如果是新版本 Git，可能要求输入 Token，见下方说明）

**如果提示输入 Token**：
1. 打开浏览器访问：`https://github.com/settings/tokens`
2. 点击 **Generate new token** → **Generate new token (classic)**
3. Note 随便填，**勾选 repo 下面的所有选项**
4. 点击 **Generate token**
5. **把页面上显示的 Token 复制下来**（很长一串字母数字）
6. 在命令提示符的密码提示处，**粘贴这个 Token**（粘贴方法：右键 → 粘贴）

---

### 4.10 验证上传成功

**打开浏览器**，访问：
```
https://github.com/你的GitHub用户名/liushenghua
```

**你应该能看到**：
- `backend/` 文件夹
- `frontend/` 文件夹
- `docs/` 文件夹
- `README.md` 文件

**恭喜！代码上传成功！**

---

## 第五阶段：注册并使用 Vercel（部署前端）

Vercel 是一个免费的网站托管平台，可以把你的网页放到网上。

### 5.1 打开 Vercel

1. 打开浏览器
2. 输入地址：`https://vercel.com`
3. 按回车

**你看到的页面**：Vercel 首页，黑色背景。

### 5.2 注册账号

1. 点击页面右上角的 **Sign Up**
2. 在注册页面，选择 **Continue with GitHub**（用 GitHub 账号登录）
3. 点击后，**会跳转到 GitHub 授权页面**
4. 在 GitHub 授权页面，点击 **Authorize**
5. 授权完成后，会跳回 Vercel

### 5.3 创建新项目

回到 Vercel 页面后：

1. 点击 **Add New...**（在页面右上角附近）
2. 选择 **Project**（第一个选项）

### 5.4 导入 GitHub 仓库

在 Import Project 页面：

1. 找到 **Select a GitHub Account** 部分
2. 在下面找到 **`liushenghua`** 仓库
3. 点击仓库右边的 **Import**

**如果没有看到仓库**：
1. 点击 **Check for repos** 链接
2. 刷新页面
3. 或者点击 **Install GitHub App** 安装 Vercel 应用

### 5.5 配置项目

Import 后进入配置页面：

1. **Framework Preset**：点击下拉框，选择 **Vue**
2. **Root Directory**：
   - 点击右边的 **Browse** 按钮
   - 进入 `frontend` 文件夹
   - 点击 **Select Folder**（或者直接点击 frontend）
3. **Build Command**：保持默认 `npm run build`
4. **Output Directory**：保持默认 `dist`

### 5.6 添加环境变量

1. 向下滚动，找到 **Environment Variables**
2. 点击 **Add**
3. 在 **Name** 框输入：`VITE_API_BASE_URL`
4. 在 **Value** 框输入：`https://liushenghua-api.up.railway.app/api/v1`
5. 点击 **Save**

### 5.7 开始部署

1. 滚动到页面底部
2. 找到 **Deploy** 按钮（绿色）
3. 点击它

### 5.8 等待部署完成

- 页面会显示部署进度
- 等待 1-3 分钟
- 完成后会显示 **Congratulations!**

### 5.9 记录前端地址

部署成功后：
1. 页面会显示你的网站地址
2. 格式类似：`https://liushenghua.vercel.app`
3. **把这个地址复制下来**

## 第六阶段：注册并使用 Railway（部署后端）

Railway 是一个提供免费云服务器的平台，可以运行 Python 程序。

### 6.1 打开 Railway

1. 打开浏览器
2. 输入地址：`https://railway.app`
3. 按回车

**你看到的页面**：Railway 首页。

### 6.2 注册账号

1. 点击页面上的 **Sign Up**
2. 选择 **Continue with GitHub**（用 GitHub 账号登录）
3. 在 GitHub 授权页面，点击 **Authorize**
4. 授权完成后，跳回 Railway

### 6.3 创建新项目

回到 Railway 页面后：

1. 点击 **New Project**（蓝色按钮）
2. 在下拉菜单中选择 **Deploy from GitHub repo**

### 6.4 选择仓库

1. 在列表中找到 **`liushenghua`** 仓库
2. 点击右边的 **Deploy**

### 6.5 等待初始部署

- Railway 会自动开始部署
- **可能会显示错误**，这是正常的，我们接下来要配置

### 6.6 创建 PostgreSQL 数据库

1. 在项目页面点击 **Add New**（在右侧边栏）
2. 选择 **Database**
3. 选择 **PostgreSQL**

**等待数据库创建完成**（约 1 分钟）

### 6.7 复制数据库连接字符串

1. 数据库创建完成后，点击 **PostgreSQL**
2. 在打开的页面找到 **Connection String**
3. 点击 **Copy** 按钮
4. **把复制的字符串记录下来**（类似：`postgresql://user:pass@host:5432/dbname`）

### 6.8 配置环境变量

回到项目设置页面：

1. 点击项目名称旁边的 **Settings**（齿轮图标）
2. 找到 **Variables** 部分
3. 点击 **New Variable**（一个一个添加）：

**添加第一个变量**：
- Name: `SECRET_KEY`
- Value: `liushenghua-secret-key-2024`
- 点击 Add

**添加第二个变量**：
- Name: `UPLOAD_DIR`
- Value: `/app/uploads`
- 点击 Add

**添加第三个变量**：
- Name: `DEBUG`
- Value: `false`
- 点击 Add

**添加第四个变量**：
- Name: `DATABASE_URL`
- Value: **粘贴刚才复制的数据库连接字符串**
- 点击 Add

**添加第五个变量**：
- Name: `PORT`
- Value: `8000`
- 点击 Add

### 6.9 配置启动命令

1. 在 Settings 页面找到 **Start Command** 或者向下滚动找到 **Custom Start Command**
2. 输入以下命令：
```
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 6.10 重新部署

1. 点击 **Deployments**（在顶部菜单）
2. 点击右上角的 **...**（三个点）
3. 选择 **Redeploy**

### 6.11 等待后端部署

- 等待 2-5 分钟
- 查看 **Logs** 可以看到部署日志
- 如果显示绿色的 **Deployed**，说明成功

### 6.12 记录后端地址

1. 点击项目名称旁边的 **Settings**
2. 向下滚动到 **Networking** 部分
3. 点击 **Add Domain**
4. 在输入框中输入你的后端域名（可以随便输入，比如 `liushenghua-api`）
5. 或者直接复制 **Public Domain** 下面的地址
6. 格式类似：`https://liushenghua-api.up.railway.app`
7. **把这个地址记下来**

## 第七阶段：连接前后端

### 7.1 更新 Vercel 前端的 API 地址

1. 打开 Vercel 项目页面
2. 点击 **Settings**
3. 找到 **Environment Variables**
4. 找到 `VITE_API_BASE_URL`，点击编辑
5. 把 Value 改成你的 Railway 后端地址，**格式要正确**：

**如果你的 Railway 地址是**：`https://liushenghua-api.up.railway.app`

**则 VITE_API_BASE_URL 应该填**：
```
https://liushenghua-api.up.railway.app/api/v1
```

6. 点击 **Save**

### 7.2 重新部署 Vercel

1. 进入 **Deployments**
2. 点击右上角的 **...**
3. 选择 **Redeploy**

### 7.3 测试网站

1. 打开你的 Vercel 前端地址
2. 点击 **注册**
3. 填写信息并提交
4. **如果能注册成功**，说明整个系统正常工作！

## 第八阶段：常见问题解决

### 问题 1：GitHub 登录失败

**解决方法**：
1. 访问 `https://github.com/settings/tokens`
2. 点击 **Generate new token**
3. 选择 **Generate new token (classic)**
4. 在 Note 里输入 `git`
5. **勾选 repo 所有选项**
6. 点击 **Generate token**
7. **复制这个 Token**
8. 之后 git push 时，密码就输入这个 Token

---

### 问题 2：Railway 部署失败

**查看日志**：
1. 在 Railway 项目页面点击 **Deployments**
2. 点击失败的部署
3. 查看 **Logs** 标签页的错误信息

**常见错误和解决**：
- `Module not found`：检查 requirements.txt 是否存在
- `Port already in use`：确保环境变量 PORT=8000
- `Database connection failed`：检查 DATABASE_URL 是否正确

---

### 问题 3：前端显示空白页

**解决方法**：
1. 按 F12 打开浏览器控制台
2. 点击 **Console** 标签
3. 如果有红色错误，截图给我看

---

### 问题 4：注册/登录不工作

**检查步骤**：
1. 按 F12 打开控制台
2. 点击 **Network** 标签
3. 点击注册按钮
4. 查看是否有红色的请求
5. 点击那个请求，查看详情
