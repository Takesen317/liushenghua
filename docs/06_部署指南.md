## 2. 前端部署（Vercel）

### 2.1 注册 Vercel 账号

1. 访问 https://vercel.com
2. 使用 GitHub 账号登录
3. 授权 Vercel 访问你的 GitHub 仓库

### 2.2 部署步骤

1. 在 Vercel Dashboard 点击 "New Project"
2. 选择 `留声画` 仓库
3. 配置项目：
   - Framework Preset: `Vue`
   - Root Directory: `./frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. 点击 "Deploy"

### 2.3 环境变量

| 变量名 | 值 | 说明 |
|--------|-----|------|
| VITE_API_BASE_URL | `https://your-backend.railway.app/api/v1` | 后端 API 地址 |

### 2.4 自定义域名（可选）

1. 在 Project Settings → Domains 添加域名
2. 配置 DNS 解析

## 3. 后端部署（Railway）

### 3.1 注册 Railway 账号

1. 访问 https://railway.app
2. 使用 GitHub 账号登录
3. 完成初始设置

### 3.2 部署步骤

#### 方式一：从 GitHub 部署

1. 在 Railway Dashboard 点击 "New Project"
2. 选择 "Deploy from GitHub repo"
3. 选择 `留声画` 仓库
4. Railway 会自动检测 Python 项目

#### 方式二：从 Docker 部署

Railway 内置支持 Dockerfile，可直接使用项目中的 `Dockerfile`

### 3.3 配置环境变量

在 Railway Project → Variables 中设置：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| DATABASE_URL | `postgresql://...` | PostgreSQL 连接字符串 |
| REDIS_URL | `redis://...` | Redis 连接字符串 |
| SECRET_KEY | `your-secret-key` | JWT 签名密钥 |
| UPLOAD_DIR | `/app/uploads` | 文件上传目录 |
| DEBUG | `false` | 调试模式 |

### 3.4 配置数据库

1. 在 Railway 中添加 "PostgreSQL" 数据库
2. Railway 会自动设置 `DATABASE_URL` 环境变量

### 3.5 配置 Redis（可选）

如果不需要 AI 异步任务，可以不配置 Redis。
Celery 在 Redis 不可用时会自动降级为同步处理。

### 3.6 挂载持久存储

1. 在 Railway 中为 Backend 服务添加 Persistent Disk
2. 挂载点设置为 `/app/uploads`

## 4. Docker 本地部署

### 4.1 启动所有服务

```bash
cd 留声画
docker-compose up -d
```

### 4.2 访问地址

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost:3000 |
| 后端 API | http://localhost:8000 |
| API 文档 | http://localhost:8000/docs |
| Nginx | http://localhost:80 |

### 4.3 查看日志

```bash
docker-compose logs -f
```

### 4.4 停止服务

```bash
docker-compose down
```

## 6. 常见问题

### Q1: Vercel 读取不到样式？

检查 `vite.config.js` 中的 base 配置：

```javascript
export default defineConfig({
  base: './',
  // ...
})
```

### Q2: API 请求跨域失败？

确保 Railway 后端配置了正确的 CORS：

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-frontend.vercel.app"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### Q3: 视频文件无法播放？

检查 `/results` 路径是否正确代理到后端。
