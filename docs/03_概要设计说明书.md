## 1. 引言

### 1.1 编写目的

本文档定义"留声画"系统的总体架构、模块划分、接口设计和数据存储方案，为详细设计和编码提供基础。

### 1.2 范围

本文档涵盖：
- 系统架构设计
- 模块划分与职责
- 接口设计
- 数据库设计
- 技术选型详细说明

### 1.3 定义与缩略语

| 术语 | 定义 |
|------|------|
| BLIP-2 | Salesforce开源的视觉-语言预训练模型 |
| Edge-TTS | 微软开源文本到语音服务 |
| MusicGen | Meta开源音乐生成模型 |
| FFmpeg | 开源音视频处理工具 |
| JWT | JSON Web Token，用户认证机制 |

### 3.2 模块职责

#### 3.2.1 前端模块

| 模块名称 | 职责 | 关键组件 |
|----------|------|----------|
| 用户认证 | 注册、登录、Token管理 | LoginForm, RegisterForm |
| 照片上传 | 拖拽上传、进度展示、预览 | UploadPanel, ProgressBar |
| 任务管理 | 发起任务、查看状态、取消 | TaskList, TaskDetail |
| 视频播放 | 视频预览、下载、分享 | VideoPlayer, ShareModal |

#### 3.2.2 后端模块

| 模块名称 | 职责 | 关键接口 |
|----------|------|----------|
| 认证模块 | 用户注册、登录、Token验证 | POST /auth/register, /auth/login |
| 文件模块 | 文件上传、下载、删除 | POST /files/upload, GET /files/{id} |
| 任务模块 | 任务创建、状态查询、结果获取 | POST /tasks, GET /tasks/{id}/status |
| 分享模块 | 生成分享链接、访问分享内容 | POST /share, GET /share/{code} |

#### 3.2.3 AI服务模块

| 模块名称 | 职责 | 输入/输出 |
|----------|------|-----------|
| BLIP-2图像分析 | 分析照片场景，生成描述 | 输入:图片 → 输出:文字描述 |
| Edge-TTS语音合成 | 将文字转为语音 | 输入:文字 → 输出:MP3音频 |
| MusicGen音乐生成 | 生成匹配场景的背景音乐 | 输入:文字描述 → 输出:WAV音频 |
| FFmpeg视频合成 | 合并照片+语音+音乐为视频 | 输入:图片+音频+音乐 → 输出:MP4 |

## 4. 接口设计

### 4.1 接口概览

| 模块 | 接口路径 | 方法 | 功能 |
|------|----------|------|------|
| 认证 | /api/v1/auth/register | POST | 用户注册 |
| 认证 | /api/v1/auth/login | POST | 用户登录 |
| 认证 | /api/v1/auth/me | GET | 获取当前用户信息 |
| 文件 | /api/v1/files/upload | POST | 上传照片 |
| 文件 | /api/v1/files/{id} | GET | 获取文件信息 |
| 文件 | /api/v1/files/{id} | DELETE | 删除文件 |
| 任务 | /api/v1/tasks | POST | 创建生成任务 |
| 任务 | /api/v1/tasks/{id} | GET | 获取任务详情 |
| 任务 | /api/v1/tasks/{id}/status | GET | 获取任务状态 |
| 任务 | /api/v1/tasks/{id}/cancel | POST | 取消任务 |
| 分享 | /api/v1/share | POST | 生成分享链接 |
| 分享 | /api/v1/share/{code} | GET | 访问分享内容 |

### 4.2 认证接口

#### POST /api/v1/auth/register

**请求体**：
```json
{
  "email": "user@example.com",
  "password": "securePassword123",
  "nickname": "用户昵称"
}
```

**响应**：
```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "user_id": "usr_xxxxx",
    "email": "user@example.com",
    "nickname": "用户昵称"
  }
}
```

#### POST /api/v1/auth/login

**请求体**：
```json
{
  "email": "user@example.com",
  "password": "securePassword123"
}
```

**响应**：
```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "access_token": "eyJhbGc...",
    "token_type": "bearer",
    "expires_in": 86400
  }
}
```

### 4.3 文件接口

#### POST /api/v1/files/upload

**请求**：multipart/form-data
- file: 文件（必填）
- description: 描述（可选）

**响应**：
```json
{
  "code": 0,
  "message": "上传成功",
  "data": {
    "file_id": "fil_xxxxx",
    "filename": "photo.jpg",
    "size": 2048576,
    "url": "/files/fil_xxxxx"
  }
}
```

### 4.4 任务接口

#### POST /api/v1/tasks

**请求体**：
```json
{
  "file_id": "fil_xxxxx",
  "style": "warm",
  "voice": "xiaoxiao",
  "music_style": "gentle"
}
```

**响应**：
```json
{
  "code": 0,
  "message": "任务创建成功",
  "data": {
    "task_id": "tsk_xxxxx",
    "status": "pending"
  }
}
```

#### GET /api/v1/tasks/{id}/status

**响应**：
```json
{
  "code": 0,

    "task_id": "tsk_xxxxx",
    "status": "processing",

    "current_step": "正在生成背景音乐",
```

## 6. AI处理流程设计

### 6.1 完整AI Pipeline

```
┌─────────────────────────────────────────────────────────────────┐
│                     留声画 AI 处理流程                            │
└─────────────────────────────────────────────────────────────────┘

用户上传照片
      │
      ▼
┌─────────────────┐
│  Step 1: 存储照片  │
│  保存原始文件     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 2: BLIP-2  │
│ 图像分析         │──── 生成场景描述
│ 生成文字caption │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 3: 风格增强  │
│ 根据style参数    │──── 调整描述情感
│ 调整描述文字     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 4: Edge-TTS│
│ 文字转语音       │──── 生成MP3音频
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 5: MusicGen│
│ 音乐生成         │──── 生成背景音乐
│ 根据场景描述     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 6: FFmpeg  │
│ 视频合成        │──── 生成MP4视频
│ 照片+语音+音乐  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Step 7: 结果存储 │
│ 保存所有产物     │
└────────┬────────┘
         │
         ▼
      任务完成
```

### 6.2 各步骤详细说明

| 步骤 | 名称 | 输入 | 输出 | 耗时(预估) |
|------|------|------|------|------------|
| 1 | 文件存储 | 原始图片 | 存储路径 | 1-3秒 |
| 2 | BLIP-2分析 | 图片 | 文字描述 | 2-5秒 |
| 3 | 风格增强 | 原始描述 | 调整后描述 | <1秒 |
| 4 | Edge-TTS | 文字 | MP3音频 | 3-8秒 |
| 5 | MusicGen | 场景描述 | WAV音频 | 10-20秒 |
| 6 | FFmpeg合成 | 图片+音频+音乐 | MP4视频 | 5-15秒 |
| **合计** | | | | **25-50秒** |

## 9. 错误处理设计

### 9.1 错误码定义

| 错误码 | 名称 | HTTP状态码 | 说明 |
|--------|------|------------|------|
| 0 | 成功 | 200 | 操作成功 |
| 1001 | 参数错误 | 400 | 请求参数不合法 |
| 1002 | 认证失败 | 401 | Token无效或过期 |
| 1003 | 权限不足 | 403 | 无权访问该资源 |
| 1004 | 资源不存在 | 404 | 文件或任务不存在 |
| 2001 | 文件上传失败 | 500 | 文件保存失败 |
| 2002 | AI处理失败 | 500 | AI模型调用失败 |
| 2003 | 视频合成失败 | 500 | FFmpeg处理失败 |
| 3001 | 服务限流中 | 429 | 请求过于频繁 |

### 9.2 错误响应格式

```json
{
  "code": 2002,
  "message": "AI处理失败，请稍后重试",
  "error_detail": "BLIP-2模型推理超时",
  "request_id": "req_xxxxx"
}
```
